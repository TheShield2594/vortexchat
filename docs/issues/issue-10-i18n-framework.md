# Add i18n framework for internationalization

**Labels:** enhancement

## Context

VortexChat currently has no internationalization infrastructure. All user-facing strings are hardcoded in English.

## Source

Previously tracked in `docs/stoat-comparison-report.md` (item #24), now deleted as all other items were completed.

## Acceptance Criteria

- [ ] Choose and integrate an i18n framework (e.g., next-intl, react-i18next)
- [ ] Extract all user-facing strings into locale files
- [ ] Add locale detection and switching UI
- [ ] Support at least English as the base locale with structure for additional languages
